# Vidlytics

**Compare. Understand. Study.** — an AI web app that compares any two YouTube videos and turns any single video into a full study kit (notes, quiz, flowchart, key terms).

🔗 Live: [vidlytics.sagargoel.shop](https://vidlytics.sagargoel.shop)

## What it does

Vidlytics has two core features, built on the same transcript-extraction + AI-analysis pipeline:

### 1. Video Compare
Paste two YouTube links and get:
- A similarity score (visual ring)
- Common topics and key differences
- An AI-written summary
- A RAG-powered chat to ask follow-up questions about either video

### 2. Study Mode
Paste one YouTube link and get:
- Structured notes, organized by section
- A 5-question quiz with instant scoring and explanations
- An auto-generated flowchart (Mermaid.js) of the video's concepts
- A key-terms glossary
- Export to Markdown or PDF

## How it works

```
YouTube URL(s)
      │
      ▼
Transcript extraction (Supadata API)
      │
      ▼
BullMQ job queue (Redis) — background processing, live progress via SSE
      │
      ▼
Gemini 2.5 Flash — structured JSON output (response schema enforced)
      │
      ▼
MongoDB (result storage) + Qdrant (transcript embeddings for RAG chat)
      │
      ▼
React UI — result tabs, quiz, flowchart render, chat
```

## Why AI is core to this, not bolted on

Every primary output is generated by Gemini with `responseMimeType: "application/json"`, so the model's output is directly structurally usable — not scraped or hand-parsed from free text:

- **Comparison analysis** — similarity scoring, topic overlap, and differences reasoned from two full transcripts in a single call.
- **Study-kit generation** — one prompt produces coherent notes, a quiz, a flowchart, and a glossary together, so they stay consistent with each other.
- **RAG chat** — transcript embeddings stored in Qdrant ground the chat's answers in the actual video content rather than the model's general knowledge.

## Running locally

```bash
# Backend
cd vidlytics-backend
npm install
# set up .env: MONGO_URI, GEMINI_API_KEY, SUPADATA_API_KEY, REDIS_HOST, REDIS_PORT, REDIS_PASSWORD
npm start

# Frontend
cd vidlytics-frontend
npm install
npm run dev
```

Redis runs via Docker (`docker-compose up -d`) — see `docker-compose.yml`.
